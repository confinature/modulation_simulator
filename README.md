# 高度なデジタル変調シミュレーター

made by Claude (Anthropic)  

ページはこちら↓  
https://confinature.github.io/modulation_simulator/  
PSK/QAM変調、ノイズ解析、BER計算、フィルタリング、アイダイアグラム、同期シミュレーションを統合した包括的な通信システムシミュレーターです。

## 概要

このシミュレーターは、デジタル通信システムの主要コンポーネントを可視化し、実践的な学習を可能にします。無線通信、衛星通信、光通信などで使用される変調方式と信号処理技術を実装しています。

## 主な機能

### 1. 多様な変調方式

#### PSK（位相偏移変調）
- **BPSK**: 2相、1ビット/シンボル - 最もロバスト
- **QPSK**: 4相、2ビット/シンボル - Wi-Fi、LTEで使用
- **8PSK**: 8相、3ビット/シンボル - 衛星通信
- **16PSK**: 16相、4ビット/シンボル - 高速伝送

#### QAM（直交振幅変調）
- **16QAM**: 16シンボル、4ビット/シンボル - LTE、Wi-Fi 6
- **32QAM**: 32シンボル、5ビット/シンボル
- **64QAM**: 64シンボル、6ビット/シンボル - LTE-Advanced、Wi-Fi 6
- **128QAM**: 128シンボル、7ビット/シンボル
- **256QAM**: 256シンボル、8ビット/シンボル - 5G、Wi-Fi 6E

QAMはPSKに振幅の変化を加えた変調方式で、より多くの情報を伝送できますが、ノイズに弱くなります。

### 2. グレイコーディング

隣接するシンボル間でビットが1ビットだけ異なるようにマッピングする技術。シンボル誤りが発生した際のビット誤り数を最小化します。

**効果：**
- ノイズによるシンボル誤りが隣接シンボルに発生しやすい
- グレイコーディングでは1ビット誤り、自然順序では複数ビット誤りの可能性

**実装：**
```
自然順序: 00, 01, 10, 11
グレイ順序: 00, 01, 11, 10
```

チェックボックスで切り替えて、コンスタレーション図のビットラベルの変化を確認してください。

### 3. ノイズとSNR制御

**SNR（Signal-to-Noise Ratio）:**
信号電力とノイズ電力の比を表し、通信品質の指標です。

- **範囲**: -5 dB 〜 30 dB
- **30 dB以上**: ほぼノイズなし（理想的な環境）
- **20 dB**: 良好な通信環境
- **10 dB**: 中程度のノイズ
- **0 dB**: 信号とノイズが同等（厳しい環境）

**受信コンスタレーション図:**
ノイズが付加された受信信号を500点プロットして表示。SNRが低いほど点が広がります。

### 4. BER（Bit Error Rate）計算

ビット誤り率は通信システムの品質を表す最重要指標です。

**理論BER曲線:**
- 各変調方式のSNRに対するBER特性をグラフ表示
- 対数スケールで1e-8 〜 0.5の範囲を表示
- QPSKは同じSNRでBPSKより約3dB劣化
- 256QAMは高SNRが必要だが、高速伝送が可能

**実用的な基準:**
- **1e-3 (0.001)**: 音声通信で許容可能
- **1e-5**: データ通信の最低ライン
- **1e-6**: 良好なデータ通信
- **1e-9**: 高品質な通信（光ファイバー）

### 5. アイダイアグラム

時間領域の信号を1シンボル周期ごとに重ね合わせて表示する解析手法。

**観察ポイント:**
- **アイの開口度**: 開いているほど信号品質が高い
- **交差点の収束**: 中央で収束するほど同期が良好
- **軌跡の太さ**: 細いほどノイズが少ない

**SNRが低いとき:**
- アイが閉じる（信号の分離が困難）
- 交差点が広がる（タイミングジッタ）
- BERが上昇

### 6. ルートロールオフフィルタ

**目的:**
- 帯域制限による符号間干渉（ISI）の抑制
- スペクトル効率の向上
- 実用的な送受信機での実装

**ロールオフ係数 α:**
- **α = 0**: 理想的だが実現不可能（矩形フィルタ）
- **α = 0.35**: 実用的なバランス（LTE）
- **α = 0.5**: 一般的な設定
- **α = 1.0**: 緩やかなフィルタ（実装が容易）

**実装:**
送信側と受信側でそれぞれルートロールオフフィルタを使用し、両者の積が完全なロールオフ特性になります。

### 7. 位相オフセットとタイミング誤差

実際の通信システムでは、搬送波の位相ずれやシンボルタイミングのずれが発生します。

**位相オフセット:**
- 搬送波再生の不完全性をシミュレート
- -180° 〜 +180°の範囲で調整
- QPSK以上では位相曖昧性が発生

**タイミング誤差:**
- シンボル同期の不完全性をシミュレート
- 0 〜 0.5の範囲（0.5 = 半シンボル分のずれ）
- アイダイアグラムで影響を確認可能

## 使い方

### 基本操作フロー

1. **変調方式を選択**
   - まずBPSKやQPSKから始めることを推奨
   - シンボル数が増えると高速だがノイズに弱くなることを確認

2. **グレイコーディングの効果を確認**
   - チェックボックスをON/OFF
   - コンスタレーション図のビットラベルの変化を観察

3. **SNRを変化させる**
   - 30 dBから徐々に下げる
   - 受信コンスタレーションの広がりを観察
   - BER値の変化を確認

4. **ビット列を変調**
   - 任意のビット列を入力、またはランダム生成
   - 時間領域波形とアイダイアグラムを確認

5. **BER曲線で性能比較**
   - 異なる変調方式でBER曲線を比較
   - 必要SNRと伝送速度のトレードオフを理解

### 学習課題

#### 初級

1. **変調方式の比較**
   - BPSK、QPSK、16QAMでコンスタレーション図を比較
   - シンボル数とビット効率の関係を理解

2. **SNRとBERの関係**
   - SNRを10 dBから20 dBに変化
   - BERが何桁改善するか観察

3. **グレイコーディングの効果**
   - SNR = 10 dBでグレイコーディングのON/OFFを比較
   - 理論的にどれだけBERが改善するか考察

#### 中級

4. **QAMの階層構造**
   - 16QAM → 64QAM → 256QAMと変化
   - 同じBERを維持するために必要なSNRの増加量を計算

5. **アイダイアグラム解析**
   - SNRを変化させてアイの開き具合を観察
   - どのSNRでアイが閉じ始めるか確認

6. **フィルタの効果**
   - ロールオフ係数を変化させて波形を観察
   - α = 0.2とα = 1.0の違いを確認

#### 上級

7. **位相オフセットの影響**
   - QPSKで位相オフセットを45°に設定
   - コンスタレーションがどう回転するか確認
   - BPSKとの違いを考察

8. **タイミング誤差とISI**
   - タイミング誤差を0.3に設定
   - アイダイアグラムの歪みを観察
   - フィルタとの組み合わせ効果を確認

9. **実システムの設計**
   - 特定のBER目標（例: 1e-6）を達成するために必要なSNR
   - 変調方式と伝送速度の最適バランスを検討

## 技術詳細

### PSK信号の数式

シンボルkに対する位相θ_k：

```
θ_k = 2π * k / M
```

送信信号（ベースバンド）：

```
s(t) = I_k + j*Q_k
ここで、I_k = cos(θ_k), Q_k = sin(θ_k)
```

### QAM信号の数式

M-QAMでは、√M × √Mのグリッド状にシンボルを配置：

```
I_k = (2i - √M + 1) * d
Q_k = (2j - √M + 1) * d
ここで、i, j = 0, 1, ..., √M-1
d は最小シンボル間距離
```

### BERの理論式

**BPSK:**
```
BER ≈ 0.5 * erfc(√(SNR))
```

**QPSK:**
```
BER ≈ 0.5 * erfc(√(SNR))  (BPSKと同等)
```

**M-QAM:**
```
BER ≈ (4(1 - 1/√M) / log₂M) * 0.5 * erfc(√(3*SNR / (M-1)))
```

### ルートロールオフフィルタ

周波数応答H(f)：

```
H(f) = 1                           (|f| ≤ (1-α)/(2T))
H(f) = cos²(πT/(2α) * (|f| - (1-α)/(2T)))   ((1-α)/(2T) < |f| < (1+α)/(2T))
H(f) = 0                           (|f| ≥ (1+α)/(2T))
```

ここで、T はシンボル周期、α はロールオフ係数。

## 実用例

### Wi-Fi (IEEE 802.11)
- **802.11a/g**: BPSK、QPSK、16QAM、64QAM
- **802.11n**: 最大64QAM
- **802.11ac**: 最大256QAM
- **802.11ax (Wi-Fi 6)**: 最大1024QAM

### LTE/5G
- **LTE**: QPSK、16QAM、64QAM
- **LTE-Advanced**: 256QAM
- **5G NR**: 最大256QAM（高SNR環境では1024QAMも検討）

### 衛星通信
- **DVB-S2**: 8PSK、16APSK、32APSK
- **INTELSAT**: QPSK、8PSK

## トラブルシューティング

**Q: BER曲線が表示されない**
A: Chart.jsの読み込みを確認してください。インターネット接続が必要です。

**Q: アイダイアグラムが表示されない**
A: ビット列を入力して「変調」ボタンをクリックしてください。

**Q: コンスタレーション図の点が少ない**
A: 受信コンスタレーションは500点をランダム生成しています。SNRを下げると広がりが見えやすくなります。

**Q: 高次QAMでコンスタレーションが見にくい**
A: 256QAMでは256個のシンボルが密集します。ズーム機能は今後の拡張で追加予定です。
